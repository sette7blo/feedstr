# Changelog

All notable changes to Feedstr will be documented here.
Versions follow [Semantic Versioning](https://semver.org): `MAJOR.MINOR.PATCH`

- **MAJOR** — breaking changes (e.g. DB schema requires migration)
- **MINOR** — new features, backwards compatible
- **PATCH** — bug fixes, visual tweaks

---

## [Unreleased]

- Simplified Feedstr settings access: removed the awkward top-right mobile `Idenstr` button and made the bottom-left sidebar chip the single Settings entry with a gear icon plus Idenstr connection status as secondary text.

- Improved zap address discovery: Feedstr now keeps the full Idenstr directory profile payload when seeding its cache, checks additional common/non-standard kind:0 zap fields (`lightning`, `lnurl`, `lnurlp`, `zapAddress`, nested zap objects), strips `lightning:` URI prefixes, and updates status copy from the overly narrow `lud16/lud06` wording to `zap address`.

- Fixed quick zaps on mobile/iOS so a short tap on the lightning bolt stays in the one-tap default-sats flow instead of opening the custom amount/comment modal. If the recipient zap address is missing, Feedstr now refreshes their kind:0 metadata in-place and only uses the modal for deliberate long-press/right-click custom zaps.

- Narrowed the iOS/mobile left drawer further to an almost-half-screen 56vw/228px layout (58vw/220px on very small screens), while preserving 44px column-close tap targets and cache-busting the stylesheet for homescreen/PWA installs.

- Added a configurable default zap amount (Settings → Zaps): tapping a note's lightning bolt now instantly zaps that amount, while holding the bolt (or right-clicking) still opens the amount/comment modal. The default is stored in Feedstr's own state store and syncs across devices. After a zap completes, the modal's only button reads Close.

- Fixed Zap profile lookup getting stuck on `checking profile for lud16/lud06…` by carrying forced zap refresh requests through the profile-fetch scheduler even when the author already has cached display metadata.

- Fixed outgoing zap discovery so opening Zap on a note force-refreshes the author's newest kind:0 profile for `lud16`/`lud06` instead of treating seeded directory profiles as complete.

- Added outgoing NIP-57 zap UI: each note has a Zap action with mobile-safe amount/comment controls, and Feedstr delegates payment to Idenstr's narrow zap endpoint so the NWC wallet secret stays in Idenstr.

- Added a close button to each sidebar column entry, with a larger always-visible touch target on mobile/iOS, so columns can be closed directly from the left pane without first switching to that column.

## [v0.1.2] — 2026-06-25

- Fixed a latent bug in the prepared-statement cache: it now re-resolves the database on every call, so a cached statement can never outlive its database handle if the data path changes at runtime. No effect on a normal single-path deployment.

## [v0.1.1] — 2026-06-24

- Performance and memory hardening. The in-memory note and engagement-count caches are now bounded (a tab left open for hours no longer grows without limit), per-event full re-sorts are replaced with sorted inserts plus O(1) dedup, profile-reference parsing is memoized, and redundant full-column scans on every relay connect are coalesced. The dashboard's static assets are gzip-compressed, HTTP keep-alive is enabled for the API, per-column cache writes are throttled (with a flush when the tab is backgrounded), prepared SQL statements are reused, and dead relays reconnect with exponential backoff instead of a flat 5s retry.

## [v0.1.0] — 2026-06-24

- Feedstr now reads Idenstr's kind:10000 mute list (`mutes:read`) and writes thread/profile/keyword mutes back through Idenstr (`mutes:write`). Muted people, threads/events, hashtags, and words are filtered through one shared predicate across normal feeds, thread replies, notification rows, and notification filter counts, so muted conversations no longer leak through Notifications. Note rows include a Mute thread action with a muted-speaker icon, and profile columns include a Mute/Muted toggle that optimistically updates Idenstr's pubkey mute list.
- Inline reply boxes now support device image uploads, matching the main composer: the reply drawer has a Photo button, uploads images through Feedstr to nostr.build with Idenstr-signed NIP-98 authorization, inserts the returned image URL into the reply textarea, and shows upload status/errors inline.
- Reply boxes no longer disappear while you are typing when new relay events/profile updates trigger feed reconciliation. Inline reply drawers are now keyed to the note they belong to and are preserved/repositioned with the note instead of being treated as stray DOM and removed.
- Fixed iOS/mobile zoom sticking after focusing the composer, reply box, or other form fields by forcing touch-device form controls to render at Safari's 16px no-auto-zoom threshold at the end of the CSS cascade, adding `maximum-scale=1` to the viewport, and cache-busting the stylesheet so existing homescreen/PWA installs pick up the fix.
- Saving the Idenstr connection no longer fails on hosts whose user is not uid 1000. The dashboard writes the token/URL back to the bind-mounted `.env`, but the container ran as a fixed uid (1000), so on a host with a different user id `.env` was unwritable and saves failed. The container now runs as your host user by default (`user: "${UID:-1000}:${GID:-1000}"`), matching Idenstr; set `UID`/`GID` in `.env` if your host user is not 1000.
- Corrected the declared Idenstr scope list: it now includes `sign:kind:5` (used to undo a like via a NIP-09 delete) and `following:write`. These were previously missing, so unliking and following only worked on an over-privileged admin token; the accurate list lets Feedstr run on a least-privilege token.
- You can now follow and unfollow people without leaving Feedstr. Open someone's profile column and use the Follow / Following toggle in the header: following is one tap, and Idenstr immediately signs the updated kind:3 contact list and broadcasts it to your private vault and public relays (no queuing, no leaving the app). Unfollow is also one tap and shows an "Undo" toast in case it was a misfire. The avatar ring updates instantly everywhere. Feedstr never authors the contact list itself — it just tells Idenstr who to add or remove, so a partial view can never clobber your follows. Requires granting Feedstr the new `following:write` scope on its Idenstr token (re-mint or edit the token in Idenstr and tick "Follow / unfollow"); until then the Follow button explains what's missing.
- Added an at-a-glance follow indicator: the avatars of people you already follow now show a purple ring, everywhere an avatar appears (feed notes, notifications, threads, the mention picker). Follow status is read from Idenstr's following list at connect, so you can tell who you follow without opening their profile.
- Made opening a profile consistent across the whole app. Tapping someone's avatar or name opens their profile column everywhere — feeds, threads, and now Notifications too — while tapping anywhere else on a note or notification opens the note/conversation. Previously the Notifications column was the odd one out: the whole row opened the note and there was no way to reach the actor's profile from it.
- Every notification is now tappable and takes you to the note it's about, not just replies. Tapping a reply still opens it in its thread; mentions now open the mentioning note in context; and reactions, reposts, and zaps open the note of yours they acted on (fetched on demand if it isn't already in view). Notifications with no note to open (e.g. a zap on your profile rather than a note) stay non-interactive. Taps on links or buttons inside a row are still ignored so they don't hijack the row click.
- Opening a column now brings it into view instead of leaving you stranded. New columns are still appended to the right of the deck (your existing column order never gets reshuffled), but the deck now smoothly scrolls and snaps to whichever column you just opened — whether it's a new hashtag/profile/custom column or an already-open one you tapped a name, avatar, or `nostr:` mention to reach. On mobile/iOS, where each column fills the whole screen, this fixes the long-standing annoyance where adding a column silently dropped you back on the first column and you had to open the menu to navigate to the one you actually opened.
- Integrated the brand mark into the UI. The header logo is now a clean rounded app-icon badge (subtle ring and depth instead of a purple halo glow), and the "Feedstr" wordmark sets its "str" suffix in the brand purple so the text and icon read as one mark — in both the desktop sidebar and the mobile top bar. Removed the purple/gold gradient strip that used to sit across the top of every modal window.
- Moved composing into a dedicated window instead of a cramped box wedged in the sidebar. Desktop gets a prominent "Compose" pencil button at the bottom of the sidebar; mobile/iOS gets a floating pencil button (bottom-right, clear of the home indicator) so you can post from anywhere without opening the menu drawer. Both open the same composer modal, which on mobile fills the screen as a sheet sized with the dynamic viewport so the keyboard never covers the text or the Post button. The textarea autofocuses on open, `Cmd/Ctrl+Enter` posts, Escape closes (after the @-mention popover), and the window closes itself once a post goes through. All the existing composer features — @-mention tagging, photo upload, and tag chips — moved across unchanged.
- Stopped feeds from flashing and avatars from strobing as relays answer. Columns used to wipe and rebuild their entire DOM on every incoming event, so each relay's batch destroyed and recreated every avatar `<img>` (forcing it to reload) and repainted the whole list — many times a second while events streamed in. Every column surface — the note timeline, the Notifications list, and open conversation threads — now reconciles in place against the existing DOM keyed by event id: already-painted rows (and their avatars) are reused and kept in order, only genuinely new ones are built. When a row's name or avatar finally loads it is patched in place — the note element, its avatar `<img>`, and its hover state all survive, and the avatar image is only swapped when the picture URL actually changed — so hovering a note no longer makes it strobe as profiles stream in. Repaints are also coalesced to one per animation frame so a burst of events from several relays produces a single smooth update instead of a flicker per relay.
- Added reply counts and a conversation view. Your own posts in the Home column now show how many replies they have (a live count on the reply action, backed by a subscription for kind:1 events that e-tag the note, deduped across relays). Tapping a note opens its thread in-column with the focused note plus its ancestors above and its direct replies listed below; replies are filtered by NIP-10 reply semantics so only true direct replies show, and they stream in live. As with notifications, reply visibility is bounded by which relays you read from.
- Widened the Notifications window. Notifications previously only fetched the last 24 hours capped at 100 events, so anything older silently never loaded; they now reach back 30 days with a 500-event cap. Visibility is still bounded by which relays carry the events.
- Likes now persist. Liking a note records it in Feedstr's DB so the heart stays filled after a refresh, and on each connect Feedstr backfills your past reactions from Idenstr's private relay (a one-shot query for your own kind:7 events, scoped to the private relay only) so previously liked notes light up too. The like itself is still the published kind:7 reaction vaulted in the private relay; the DB only remembers which notes to fill the heart on. Completeness depends on what the private relay has vaulted, so likes made on other clients that never reached it won't show.
- Reworked threads and reply context to read like a normal timeline instead of a dialog. Tapping a parent/reply now opens the conversation in-place inside that column with a back arrow in the header, instead of a floating modal window. Removed the uppercase PARENT/ROOT/SELECTED role labels and the gold gradient highlight in favour of a single thread line connecting the avatars and a faint highlight on the focused note. Demoted the large bordered "replying to" card in feeds to a quiet one-line "Replying to @name" cue. Made the whole note tappable to open its thread (the Damus/Nostur pattern), ignoring taps on links, buttons, and text selections.
- Gave Feedstr its own SQLite metadata DB (`node:sqlite`, persisted under `/data`), so the app no longer relies on browser storage for the state it owns. Column config now lives server-side (auto-migrated once from the old `feedstr:columns` browser entry), and observed notes are cached per column so feeds are warm on the next visit. This matches the stack storage boundary: signed events stay in Idenstr's private relay, everything Feedstr owns stays in its DB. New `/data` volume — back it up alongside your `.env`.
- Added a photo button to the composer: selecting an image from the device uploads it through Feedstr to the nostr.build NIP-96 free upload API with NIP-98 authorization from Idenstr, then inserts the returned media URL into the note before publishing. Hardened the upload flow so it cannot start overlapping uploads, rejects images over nostr.build's 20 MB free limit immediately, times out stuck uploads after 90 seconds, sends mobile-friendly fixed-length JSON responses, and shows the returned URL visibly below the composer as well as in the note box.
- Polished the UI to match Idenstr: reworked the composer so tagging a follow inserts a readable `@Name` (instead of a raw `nostr:nprofile…` blob) with removable "Tagging" chips below the box, resolved to proper Nostr `p`-tags on publish; removed the duplicate compose hint; and stopped tagging from overwriting the relay status line. Consolidated the four redundant sidebar connection elements into a single status/links footer at the bottom of the sidebar — an Idenstr connection chip (opens the settings modal) grouped with the live relay/follow counts. Unified column empty states, tightened column/note headers and action buttons, and normalized spacing, pills, and focus styles across the app.

- Added Idenstr connection diagnostics and setup guidance for URL + scoped token linking.
- Switched Feedstr publishing to Idenstr's Phase 1 REST signing flow instead of admin publish endpoints or local key custody.
- Feedstr now discovers Idenstr stack topology, including the private relay URL, from Idenstr's API instead of asking for Feedstr bind or relay wiring in the app-linking UI.
- Made Idenstr's private relay an editable, testable field in the connection panel: enter the relay's LAN or Tailscale address, run a one-click Test that opens a WebSocket from the browser (exactly how the feed reads it) and reports reachability/latency and whether it speaks Nostr, then Save it back to `.env`. Renamed `FEEDSTR_LOCAL_RELAY` to `FEEDSTR_PRIVATE_RELAY_URL` to match Idenstr's private-relay naming (update your `.env`).
- Restyled the Feedstr shell and Idenstr connection modal to match Idenstr's sovereign dark panel theme, pressed-button feedback, topology cards, and terminal-style status output.
- Added an iOS-friendly mobile layout with a compact safe-area top bar, off-canvas menu drawer, full-width feed columns, and mobile note spacing so the sidebar no longer consumes the screen.
- Fixed iOS mobile drawer blur by removing backdrop-filter from the menu scrim, lifting the drawer above the scrim, and using GPU-stable translate3d transforms.
- Fixed mention/notification avatars by fetching author kind:0 profiles on demand when cached profiles are missing pictures, rerendering visible columns as profiles arrive, and allowing notification event kinds to render in the Notifications column.
- Unified Feedstr mentions into the Notifications column with typed icon filter chips, notification-specific rows, and zap sender/amount parsing.
- Improved notification avatar recovery by retrying kind:0 profile lookups when avatar metadata is missing or an image URL fails to load.
- Fixed notification rows disappearing by fanning reply/mention/repost/reaction/zap subscriptions out to all connected relays instead of a sampled relay subset.
- Replaced Feedstr favicon, Apple touch icon, PWA icons, and header logos with the new purple `fd` app icon.
- Added inline media rendering for note content so image URLs show as image previews while raw image links are hidden, and non-image URLs show as compact link cards.
- Added embedded Nostr event previews for `nostr:note`, `nostr:nevent`, and `nostr:event` references, including relay lookup for referenced events.
- Added inline parent context previews for reply notes, using NIP-10 `reply` tags or the last `e` tag fallback and fetching missing parent notes from relays.
- Added a conversation view for reply parent cards: clicking a parent preview opens the thread context, shows parent/root notes above, and highlights/scrolls to the selected reply.
- Added Nostr person/tag rendering for `npub`, `nprofile`, `nostr:nprofile`, and indexed `#[n]` p/e/t tags so people and tags display as readable chips instead of raw references, including profile refresh/rerendering for referenced people and profile-discovery relay fallback.
- Fixed Following column avatar flicker by rerendering profile rows only when incoming kind:0 metadata changes visible display fields.
- Added a composer tag picker that inserts NIP-19 profile references from the following directory and publishes matching `p` tags with new notes.
- Replaced the composer tag picker with inline `@` autocomplete: typing `@` opens a filtered follow list, arrow/Enter or click inserts the NIP-19 profile reference, and posts still include matching `p` tags.
